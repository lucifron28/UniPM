# Evidence Work Instructions

- Read `reference/evidence/README.md` before adding or changing evidence.
- Follow its naming, front matter, evidence-level, status, and sanitization
  rules.
- Link claims to exact commits, generated artifacts, tests, Git history, or
  source paths. Use factual language and state unavailable verification.
- Do not use promotional or manuscript-style overclaiming.
- Redact secrets, credentials, connection strings, private endpoints, personal
  data, prompts, token maps, provider payloads, and vectors.
- Distinguish synthetic results from production evidence and deterministic
  provider orchestration from real semantic model-quality evidence.
- Never rewrite an approved record to make later results look better. Create a
  new record with a new ID for a new experiment or verification run.
- Update `INDEX.md` whenever an evidence record is added or superseded.
